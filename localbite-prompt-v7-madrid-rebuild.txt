STEP 1 — Write search plan

Before running any searches, write a plan to OUTPUT_SEARCH_PLAN.

MANDATORY SELF-CONFIRMATION — write this as the first line
of OUTPUT_SEARCH_PLAN before anything else:
"MANDATORY REQUIREMENTS CONFIRMED: year range distribution
across [YEAR_RANGE], cross-validation Phase 1B after main
searches, audit file required, 10 results per query (hard limit)."
If you cannot confirm all four, stop and report which
requirement cannot be met before proceeding.

The search plan has two parts:

PART A — NEIGHBOURHOOD COVERAGE (non-negotiable)
If NEIGHBOURHOOD_TIERS is defined in Part 1, use those exact
areas and tier assignments. Do not substitute or add areas.

If NEIGHBOURHOOD_TIERS is not defined, run one search query
per neighbourhood or neighbourhood group listed in
GEOGRAPHIC_BOUNDARY_INCLUDE. Adjacent neighbourhoods with
overlapping food writing scenes may be grouped into one query.
Every neighbourhood must appear in at least one query.

Run neighbourhood queries in all relevant SEARCH_LANGUAGES.
For a bilingual city: run each neighbourhood in both languages
(2 queries per neighbourhood or group). For a trilingual city:
run in all 3 languages (3 queries per neighbourhood or group).
For a single-language city: 1 query per neighbourhood or group.

If NEIGHBOURHOOD_TIERS is defined with 3 tiers, structure
queries as follows per tier:
- Tier 1 (central/tourist): 1 EN + 1 local language per area
- Tier 2 (residential): 1 EN + 1 local language per area
- Tier 3 (emerging): 1 EN + 1 local language per area
This produces 2 queries per named area × number of areas.

PART B — CROSS-CUTTING ANGLES (run after all neighbourhood queries)
These angles cover what neighbourhood queries miss:

  Cross-1 — City-wide EN: 2 queries
    Vary register — one editorial guide style, one personal
    recommendation style. Do not use the same phrasing as
    neighbourhood queries.

  Cross-2 — City-wide local language: 3 queries
    Use 3 meaningfully different registers: formal food
    criticism, casual neighbourhood recommendation, food
    journalism / press guide.

  Cross-3 — Generic writer-first: 2 queries
    Open discovery queries. Do not name specific writers.
    1 EN + 1 local language.

  Cross-4 — Named awards: 3 queries
    Use award names from NAMED_AWARDS exactly. One query per
    award. Target articles about results and winners — not
    the award body itself. If NAMED_AWARDS is blank, run
    generic award queries: 1 EN + 1 local language + 1 Michelin.

  Cross-5 — Recent openings 2025-2026: 2 queries
    1 EN + 1 local language. Target new openings specifically.

  Cross-6 — Vocabulary-specific local language: 2 queries
    Use vocabulary from LOCAL_FOOD_VOCABULARY. Target the
    casual neighbourhood register city-wide queries miss.
    If LOCAL_FOOD_VOCABULARY is blank, skip this angle.

  Cross-7 — Cuisine and community: 2 queries
    Target food communities whose writing is not captured
    by generic city-wide queries. Use specific cuisine types
    or cultural communities relevant to this city.
    If not applicable, skip this angle.

  Cross-8 — Publication-category local language: 2 queries
    Target categories of local publication by descriptor only.
    Do not name specific publications.
    1 independent food magazine / revista gastronómica
    1 newspaper gastronomy supplement

SOURCE QUALITY RULES:

✓ Personal single-author site — About page byline sufficient

✗ No named author anywhere — reject regardless of publication
✗ Pre-2023 date (verify from fetched text, not updated label)
✗ Booking sites, hotel sites, sponsored content
✗ Covers whole country or region not this city specifically
✗ KNOWN_PROBLEMATIC_URLS
✗ Social media primary sources without fetchable articles
✗ Articles that read as promotional or lack specific critical detail

STEP 2 — Assign confidence tier

BOTH-POOL DEFINITION (strict):
A restaurant qualifies as both-pool ONLY if it appears in:
- At least one source in language pool A, AND
- At least one source in language pool B, AND
- The two sources are from DIFFERENT publishers with no shared
  ownership, brand relationship, or licensing arrangement.
Same-publisher cross-language editions do NOT qualify as
both-pool even if they have different editorial teams.

TIER A — Auto-approve:
- 2 or more independent sources, OR
- Both-pool (using strict definition above)

For single-language cities (SINGLE_LANGUAGE_CITY: YES):
- Tier A = legacy/institutional + independent/freelance source

TIER B — Review recommended:
- Single primary or independent source, good fetch quality,
  quote ≥15 words with ≥1 specific detail
- Secondary single source, good fetch quality, quote ≥15 words
  with ≥2 specific details

TIER C — Auto-reject:
- Failed fetch, single source
- Quote under 15 words or no specific details
- Secondary single-source with fewer than 2 specific details
- Outside GEOGRAPHIC_BOUNDARY_INCLUDE
- Low-confidence flag from extraction
- Failed traceability check

SOURCE CONCENTRATION CAP:
If any single source contributes more than 30% of total
(Tier A + Tier B) entries as single-source Tier B:
Auto-reject lowest-confidence entries from that source until
within cap. Report in AUTO-REJECTED section.

- open_status_check entries — flag but do NOT run closure
