After writing the search plan to OUTPUT_SEARCH_PLAN, run
the prior knowledge check script:

  python3 localbite-prior-knowledge-check.py \
    localbite-madrid-v7-search-plan.txt

This script checks every query for proper nouns and named
entities not present in Part 1 variables.

If the script exits with code 2 (definite violations):
  - Do NOT proceed to searches
  - Rewrite the flagged queries to remove named entities
  - Re-run the check until exit code is 0 or 1
  - Log: [prior-knowledge-check] VIOLATIONS FOUND — rewrote
    queries [list labels] before proceeding

If the script exits with code 1 (review flags only):
  - Inspect each flagged term
  - If the term is a generic food/writing term → proceed
  - If the term is a named entity not in Part 1 → rewrite
  - Log: [prior-knowledge-check] REVIEW FLAGS — inspected,
    [N] terms permitted, [N] queries rewritten

If the script exits with code 0 (all clear):
  - Log: [prior-knowledge-check] PASSED — all queries clean
  - Proceed to Phase 1 searches immediately
